Controlling Access to the Open Petra Web Server

You can choose whether to respond to 'remote' requests for information from your web sites from other computers. You might do this if you want to set up a local office intranet. Allowing remote connections is more of a security risk. You need to make sure that the folders that make up your web site only have read and execute access for a 'standard user'.

The way that you control access to your computer is one of the following

  • If you launch the Open Petra Web Server by specifying the site details on the command line, you will need to use the /r switch if you wish to allow remote connections. The default behaviour is to reject remote connections.
  • If you launched the Open Petra Web Server with no command line arguments you can accept remote connections by checking the Allow Remote Connections item on the Options menu. The options menu is accessible both from the main GUI and from the System Tray. By default, remote connections are not accepted.
